Load and Explore a Dataset

  • ID: CDI-DS-EXPLORE-002
  • Type: Core Lesson
  • Audience: Beginner / Intermediate
  • Theme: Exploration before interpretation

Analytical work does not begin with modeling.

It begins with looking carefully at the data.

Before fitting models or drawing conclusions, we need to understand: - what the dataset contains - how variables are structured - whether values look plausible - and whether early patterns deserve closer attention

In CDI, this stage is not a formality. It is where we begin to build analytical judgment.

In this lesson, you will:

  • load the Iris dataset into a pandas DataFrame
  • inspect its structure and column types
  • generate summary statistics
  • create modern exploratory plots
  • save the dataset for reuse in later lessons

Why this step matters

A dataset can look clean at first glance and still contain issues that affect later analysis.

Early exploration helps you answer practical questions such as:

  • How many rows and columns do I have?
  • Which variables are numeric and which are categorical?
  • Do any values look unusual?
  • Are there visible group differences?
  • What should I examine more carefully next?

This is the beginning of defensible analysis.

Load the dataset

We will use the Iris dataset from scikit-learn, convert it to a pandas DataFrame, and standardize the column names.

Code 
import pandas as pd
from sklearn.datasets import load_iris

iris = load_iris()

df = pd.DataFrame(iris.data, columns=iris.feature_names)
df["species"] = pd.Categorical.from_codes(iris.target, iris.target_names)

df.columns = ["sepal_length", "sepal_width", "petal_length", "petal_width", "species"]

df.head()
sepal_length sepal_width petal_length petal_width species
0 5.1 3.5 1.4 0.2 setosa
1 4.9 3.0 1.4 0.2 setosa
2 4.7 3.2 1.3 0.2 setosa
3 4.6 3.1 1.5 0.2 setosa
4 5.0 3.6 1.4 0.2 setosa

What happened here?

  • iris.data contains the numeric measurements
  • iris.target contains the species labels encoded as integers
  • pd.Categorical.from_codes() converts those encoded values into readable species names
  • the columns are renamed to snake_case so later code stays clean and consistent

Save the dataset for future lessons

We will keep reusable datasets inside a local data/ folder.

Code 
from pathlib import Path

Path("data").mkdir(exist_ok=True)
df.to_csv("data/iris.csv", index=False)

print("Saved dataset to: data/iris.csv")
Saved dataset to: data/iris.csv

Saving the dataset at this point helps later lessons stay reproducible and consistent.

Exploratory visualizations

We will use modern plotting with seaborn and matplotlib, without CDI theme dependencies.

These plots are not yet final presentation graphics. Their purpose is to help us inspect the data and notice patterns worth explaining.

Distribution of numeric features

Code 
import seaborn as sns
import matplotlib.pyplot as plt

sns.set_theme(style="whitegrid", context="notebook")

df_long = df.melt(
    value_vars=["sepal_length", "sepal_width", "petal_length", "petal_width"],
    var_name="feature",
    value_name="value"
)

g = sns.displot(
    data=df_long,
    x="value",
    col="feature",
    col_wrap=2,
    bins=12,
    height=3.6,
    aspect=1.15,
    facet_kws={"sharex": False, "sharey": False}
)

g.set_titles("{col_name}")
g.set_axis_labels("Value", "Count")
g.fig.suptitle("Iris — Distribution of Numeric Features", y=1.03)

plt.show()

Interpretation

These histograms help us see:

  • the overall spread of each feature
  • whether values are concentrated or dispersed
  • whether some variables may contain overlapping or separated groups

A single histogram does not separate species, but it gives a first impression of the measurement ranges.

Boxplots by species

Code 
ffig, ax = plt.subplots(figsize=(8, 5))

# Boxplot (soft blue palette)
sns.boxplot(
    data=df,
    x="species",
    y="petal_length",
    width=0.5,
    fliersize=0,
    boxprops=dict(facecolor="darkblue", alpha=0.8),
    medianprops=dict(color="steelblue", linewidth=1.8),
    whiskerprops=dict(color="steelblue"),
    capprops=dict(color="steelblue"),
    ax=ax
)

# Points (warm contrast)
sns.stripplot(
    data=df,
    x="species",
    y="petal_length",
    color="steelblue",
    alpha=0.8,
    size=4.5,
    jitter=0.22,
    ax=ax
)

ax.set_title("Petal Length by Species")
ax.set_xlabel("Species")
ax.set_ylabel("Petal Length")

plt.show()

Interpretation

This plot compares distributions across species directly.

You should already see that petal length differs strongly by species, which suggests this feature may be especially informative in later analyses.

Scatter plot: sepal length vs petal length

Code 
fig, ax = plt.subplots(figsize=(8, 5.5))

sns.scatterplot(
    data=df,
    x="sepal_length",
    y="petal_length",
    hue="species",
    s=70,
    alpha=0.8,
    ax=ax
)

ax.set_title("Sepal Length vs Petal Length")
ax.set_xlabel("Sepal Length")
ax.set_ylabel("Petal Length")

plt.show()

Interpretation

Scatter plots help us inspect relationships between two variables at the same time.

Here, the species begin to separate into visible clusters, especially when petal length is involved. That is a strong exploratory signal that some measurements carry more discriminatory information than others.

Pairwise relationships

Code 
g = sns.pairplot(
    df,
    hue="species",
    corner=True,
    diag_kind="hist",
    plot_kws={"alpha": 0.7, "s": 45}
)

g.fig.suptitle("Iris — Pairwise Relationships by Species", y=1.02)

plt.show()

Interpretation

The pairplot gives a broader view of the dataset:

  • which variable pairs show separation
  • where clusters overlap
  • whether some variables appear more useful for distinguishing species

This kind of overview is valuable before choosing formal models.

Create a simple derived feature

Feature creation is often part of exploration.

Here we create a new variable called petal_area as a simple combination of petal length and petal width.

Code 
df["petal_area"] = df["petal_length"] * df["petal_width"]

CDI Insight

Exploration is not just about making plots.

It is about learning how the dataset behaves before you ask it to support conclusions.

A responsible analyst does not rush from data loading to modeling. They pause, inspect, compare, question, and only then move forward.

That habit is one of the foundations of reliable analysis.
